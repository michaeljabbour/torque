Torque

Write constraints for an intelligence.

Declare what you want (Context), why you want it (Intent), and how safe it should be (Behavior). The kernel reads your Intent and compiles it into a REST API, an Agent Tool Schema, and real-time execution streams — simultaneously. You write the contract. The framework delivers the infrastructure.

Why Torque

Controlled force around a single axis.

Torque applies mechanical advantage to application development. You define the contract. The kernel multiplies it into infrastructure. No brute force — just leverage.

📋
Manifest-First

A manifest, a logic file, and optionally an Intent triplet. The kernel provisions your database, registers your API, wires events, enforces isolation, and auto-generates Agent Tool Schemas.

🔒
Runtime Isolation

Each bundle gets its own scoped data access and a coordinator that only reaches declared dependencies. Cross a boundary you didn’t declare? The kernel stops you. Not a warning — a wall.

🤖
Intent-Driven

Stop writing controllers. Declare Intents with success criteria and behavioral guardrails. The kernel compiles each Intent into a dual-interface API (humans and agents use the same endpoint), an Agent Tool Schema, and real-time WebSocket streams. Safety is mathematical — requireHumanConfirmation physically gates dangerous tools at runtime.

Intent-Driven Design

Intent is the new controller.

Traditional frameworks have routes and controllers. Torque has Intents — first-class objects that compile down to REST endpoints, Agent Tool Schemas, and reactive streams simultaneously.

📡
Context (What)

Data boundary + vectorize for semantic indexing via VORM (Vector-Object Relational Mapping). The agent sees precisely what it needs — no more, no less.

🛡️
Behavior (How)

Mathematical safety, not prompt engineering. allowedTools restricts what the agent can call. requireHumanConfirmation physically gates dangerous actions at the runtime layer.

🎯
Intent (Why)

Goals + successCriteria. The kernel evaluates "done" — you never write if/else controller logic again. One Intent compiles into three manifestations automatically.

Three Manifestations

Define once. Compile to three.

🌐
REST API

POST /api/intents/kanban/CreateCard — standard JSON for your React UI. The kernel skips the LLM for simple payloads. Same endpoint serves humans and agents.

🤖
Agent Tool Schema

JSON schema passed to the LLM at runtime. The model knows the data shape, constraints, and success criteria natively. Works with Claude, GPT, or any provider.

⚡
Reactive Streams

WebSocket events via HookBus. Real-time UI rendering: Agent thinking… → Tool invoked → Success criteria met → Done. Full observability, no black box.

Intelligence Contracts

Smart contracts for cognition.

Ethereum built smart contracts to secure financial intent. Torque builds Intelligence Contracts to secure cognitive intent.

📜
The Contract

Intent + Behavior = immutable execution constraints. You don't trust the AI — you trust the Intelligence Contract wrapped around it. allowedTools is physical, not linguistic.

📒
The Ledger

The HookBus emits every agent action as a structured event — a private behavioral blockchain. Full audit trail: intent in, state out. As traceable as Etherscan.

🧱
Composable Legos

Bundles are your network's smart contracts. The Registry enforces dependency graphs. bundle.lock tracks evolution. Stack safely, compose fearlessly.

In Practice

One manifest. One logic file. That's a complete feature.

See how Torque transforms your bundle declaration into a fully operational API with schema, events, and isolation boundaries baked in.

bundles/todos/manifest.yml
bundle:       todos
version:      "1.0.0"
description:  "Full CRUD todo management"

schema:
  todos:
    columns:
      id:
        type:           integer
        primaryKey:    true
        autoIncrement: true
      title:
        type:    string
        notNull: true
      completed:
        type:    boolean
        default: false
      created_at:
        type:    datetime
        default: now

events:
  emits:
    - todo.created
    - todo.completed
  listens:
    - auth.user.created

interfaces:
  - name:    TodoRepository
    methods:
      - findAll(): Promise<Todo[]>
      - findById(id): Promise<Todo>
      - create(data): Promise<Todo>

routes:
  - GET    /api/todos
  - POST   /api/todos
  - PUT    /api/todos/:id
  - DELETE /api/todos/:id
bundles/todos/logic.js
export default class TodosBundle {

  constructor(context) {
    this.db     = context.get('DataLayer');
    this.events = context.get('EventBus');
  }

  interfaces() {
    return {
      TodoRepository: {
        findAll: () =>
          this.db.query('todos').all(),

        findById: (id) =>
          this.db.query('todos')
            .where({ id }).first(),

        create: async (data) => {
          const todo = await this.db
            .query('todos').insert(data);
          await this.events
            .emit('todo.created', todo);
          return todo;
        }
      }
    };
  }

  routes() {
    const repo = this.interfaces().TodoRepository;
    return {
      GET    '/api/todos':    () => repo.findAll(),
      POST   '/api/todos':    (data) => repo.create(data),
    };
  }
}
bundles/notifications/logic.js
export default class NotificationsBundle {

  constructor(context) {
    this.events = context.get('EventBus');
  }

  listen() {
    // Automatically subscribed via manifest
    this.events.on('todo.created', async (payload) => {
      await this.sendNotification(
        `New todo created: ${payload.title}`
      );
    });
  }

  sendNotification(msg) {
    // Send to Slack, email, push, etc.
    console.log('Notification:', msg);
  }
}

Getting Started

From zero to API in four commands.

Torque is designed to feel natural. Create, scaffold, boot, and deploy — that's it.

1
Create App
torque new my-app

Scaffold your application with the standard directory structure, configurations, and kernel setup.

2
Generate Bundle
torque generate todos

Create a new bundle with manifest.yml and logic.js. Choose your schema columns with the interactive prompt.

3
Boot Kernel
torque dev

Start the development kernel. It reads all manifests, provisions databases, wires events, and mounts routes.

4
API Live
curl localhost:9292

Your full-stack API is running on port 9292 with OpenAPI docs, health checks, and introspection endpoints built in.

First Principles

Why composition beats configuration.

Torque is built on ideas proven by Unix, LEGO, and the Linux kernel. Here's the thinking behind the framework.

🧩
The LEGO Principle

Small, self-contained pieces with shared contracts snap together. A brick doesn't care what it becomes — spaceship, castle, or something new. The connector is the same on every brick. In Torque, the connector is the manifest.

🔧
Mechanism, Not Policy

The kernel provides capability. Bundles make decisions. This is the Unix philosophy applied to full-stack apps: a thin orchestrator that reads contracts, provisions services, and enforces boundaries — then gets out of your way.

📈
Composition Scales Linearly

Configuration grows combinatorially — 10 flags with 2 states = 1,024 combinations. Composition grows linearly — 10 modules = 10 independent choices. This is why composability wins at scale, and why Torque uses bundles instead of feature flags.

👁
Event-First Observability

Events are not a logging afterthought — they're the primary communication surface. Every bundle publishes and subscribes through the EventBus. If it matters, it's an event. If you need to react, you subscribe. This is how bundles stay isolated while staying connected.

🎯
Intent as First-Class Object

Controllers tell a machine what to do. Intents tell it what "done" looks like. The framework compiles your Intent into a REST endpoint, an Agent Tool Schema, and real-time streams — you define constraints, not instructions. Programming at the level of "why."

📜
Intelligence Contracts

Like Ethereum smart contracts enforce financial transactions, Torque Intelligence Contracts enforce cognitive transactions. requireHumanConfirmation is a multisig gate. The HookBus is a behavioral ledger. Safety is mathematical, not linguistic.

The Pattern Progression

Rails made convention over configuration mainstream. ASP.NET Core made dependency injection mainstream. Torque makes Intent-Driven runtime intelligence mainstream — each generation makes the boundary between human and machine harder to violate.
